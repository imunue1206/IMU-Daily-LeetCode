## 📋 项目简介

IDEA本地LeetCode执行器是一个专为IntelliJ IDEA用户设计的LeetCode刷题辅助工具。它允许你在IDEA本地环境中直接执行LeetCode题目，自动记录每一次的解题过程（包括运行/调试模式、执行时间、连续尝试次数等），并生成详细的训练分析报告，帮助你全面了解自己的学习进度和能力提升轨迹。

### 🎯 设计理念

- **本地优先**：无需浏览器，在熟悉的IDEA环境中完成所有刷题操作
- **过程记录**：不仅记录结果，更记录整个解题过程的每一步尝试
- **智能分析**：通过数据分析揭示你的学习模式和提升空间
- **无缝集成**：完美融入IDEA开发流程，提供流畅的开发体验

### ✨ 核心功能

- **🎨 IDEA深度集成**：专为IntelliJ IDEA设计，完美融入开发环境
- **💻 本地题目执行**：无需访问LeetCode网站，直接在IDEA中运行和调试题目
- **📝 解题过程记录**：自动记录每一次执行细节，包括：
  - 运行/调试模式区分
  - 精确的执行时间戳
  - 连续尝试次数统计
  - 题目难度和名称
- **📊 智能训练分析**：基于执行记录生成分析报告，帮助你：
  - 追踪训练进度
  - 识别薄弱环节
  - 优化学习策略
- **🏋️ 难度梯度管理**：支持EASY、MEDIUM、HARD三级难度分类和筛选
- **🔄 动态执行引擎**：通过反射机制灵活加载和执行指定题目的解决方案

## 🚀 IDEA中快速开始

### 环境要求

- IntelliJ IDEA (推荐2021.3+)
- JDK 8+ 开发环境
- Maven 3.x 构建工具

### IDEA中操作步骤

1. **导入项目**
   - 打开IDEA，选择`File` → `Open`
   - 选择项目根目录下的`pom.xml`文件
   - 选择`Open as Project`，等待Maven依赖加载完成

2. **运行指定题目**
   - 打开`src/main/java/problem/Run.java`
   - 修改题目编号：

   ```java
   // 在Run.java中修改题目编号
   ExecuteUtil.runWithNumber(392);  // 执行题目392
   ```

3. **执行题目**
   - 点击IDEA工具栏中的▶️ 运行按钮，或使用快捷键`Shift + F10`
   - 或使用🐛 调试按钮，或快捷键`Shift + F9`进行调试

4. **查看执行记录**
   - 执行后，系统会自动在项目根目录生成`execution_statistics.txt`文件
   - 记录内容包含：题目编号、难度、名称、执行时间、运行模式、连续执行次数

### 本地记录文件格式

```
# execution_statistics.txt 记录格式
题目编号|难度|题目名称|执行时间|运行模式|连续执行次数

# 示例记录
392|EASY|Is Subsequence|2024-01-01 12:00:00|RUN|1
392|EASY|Is Subsequence|2024-01-01 12:01:30|DEBUG|2
```

## 🔧 IDEA开发指南

### 在IDEA中添加新题目

1. **创建新类**
   - 右键点击`problem.resolution`包
   - 选择`New` → `Java Class`
   - 按照`Problem{题号}.java`的命名规范命名（如：`Problem123.java`）

2. **实现抽象方法**
   - 让新类继承`LeetCodeProblem`抽象基类
   - 实现三个核心方法：
     - `run()`: 解题逻辑实现
     - `getTitle()`: 返回题目名称
     - `getDifficulty()`: 设置题目难度（EASY/MEDIUM/HARD）

3. **IDEA智能提示**
   - 利用IDEA的代码补全功能快速生成方法骨架
   - 使用IDEA的调试工具进行解题思路验证

示例：

```java
package problem.resolution;

public class Problem123 extends LeetCodeProblem {
    @Override
    public void run() {
        // IDEA中编写和调试解题逻辑
        // 支持断点、变量查看、表达式求值等所有IDE调试功能
    }

    @Override
    protected String getTitle() {
        return "Best Time to Buy and Sell Stock III";  // 题目名称
    }

    @Override
    protected Difficulty getDifficulty() {
        return Difficulty.HARD;  // 题目难度
    }
}
```

## 📊 本地记录与分析功能

### 执行记录格式

系统自动将每一次执行记录保存到`execution_statistics.txt`文件中，采用结构化的管道分隔符格式：

```
题目编号|难度|题目名称|执行时间|运行模式|连续执行次数
```

### 记录内容详解

| 字段 | 说明 | 分析价值 |
|------|------|---------|
| 题目编号 | LeetCode题目唯一标识 | 跟踪完成的题目范围 |
| 难度 | EASY/MEDIUM/HARD | 分析难度分布和挑战偏好 |
| 题目名称 | 完整题目名称 | 直观了解练习的具体题目 |
| 执行时间 | 精确到秒的时间戳 | 分析学习时间分布和时长 |
| 运行模式 | RUN/DEBUG | 识别需要调试的困难题目 |
| 连续执行次数 | 同一题目的连续尝试次数 | 发现需要反复练习的题目 |

### 示例记录

```
392|EASY|Is Subsequence|2024-01-01 12:00:00|RUN|1
392|EASY|Is Subsequence|2024-01-01 12:01:30|DEBUG|2
15|MEDIUM|3Sum|2024-01-01 14:20:00|RUN|1
15|MEDIUM|3Sum|2024-01-01 14:25:00|DEBUG|2
15|MEDIUM|3Sum|2024-01-01 14:30:00|DEBUG|3
```

## 🎯 IDEA专属功能特点

### 智能执行模式检测

系统深度集成IDEA调试器，自动识别运行模式：

- **RUN模式**：正常执行，代表对题目解法的熟练掌握
- **DEBUG模式**：调试执行，代表正在探索解题思路或排查问题

通过区分这两种模式，可以更准确地分析你的训练质量和对不同题目的掌握程度。

### 解题过程追踪

不仅仅记录执行结果，更记录完整的解题过程：

- **连续执行次数**：同一题目连续执行的次数，反映掌握难度
- **时间分布**：每次执行的时间戳，帮助分析学习习惯和效率
- **难度趋势**：不同难度题目的练习比例，反映训练挑战度

### IDEA体验增强

- **一键运行/调试**：直接使用IDEA的运行和调试功能
- **代码补全支持**：利用IDEA强大的代码补全和智能提示
- **版本控制集成**：与Git无缝集成，便于保存解题过程的版本历史
- **快捷键操作**：支持IDEA的所有快捷键，提供流畅的开发体验
